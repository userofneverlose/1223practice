## 1. Аутентификация и авторизация

В проекте реализованы два механизма защиты доступа к чувствительным данным.

| Механизм | Реализация | Уровень защиты | Где используется |
|----------|------------|----------------|------------------|
| **Basic Auth** | Логин/пароль через заголовок `Authorization` | 🟡 Учебный | Эндпоинты `/admin/*` |
| **API Key** | Bearer Token в заголовке `Authorization` | 🟡 Учебный | Эндпоинты `/mobile/*` |
| **Ролевая модель** | Разделение на Администратора и Клиента | 🟢 Есть | Все защищённые роуты |

### 🔴 Рекомендации для Production:
1.  Заменить Basic Auth на **JWT-токены** ( Stateless аутентификация).
2.  Пароли хранить только в хэшированном виде (алгоритм `bcrypt` или `argon2`).
3.  Реализовать полноценную систему пользователей с базой данных `Users`.
4.  Добавить двухфакторную аутентификацию (2FA) для администраторов.

## 2. Защита от типовых уязвимостей (OWASP Top 10)

Проект защищён от большинства распространённых веб-угроз благодаря использованию современных фреймворков (FastAPI, SQLAlchemy, Pydantic).

| Уязвимость | Механизм защиты | Статус | Комментарий |
|------------|-----------------|--------|-------------|
| **SQL-инъекции** | SQLAlchemy ORM (параметризованные запросы) | 🟢 **Защищено** | Ручные SQL-запросы отсутствуют |
| **XSS (Межсайтовый скриптинг)** | FastAPI автоматически экранирует JSON | 🟢 **Защищено** | Данные возвращаются только в JSON |
| **CSRF (Подделка запросов)** | CORS ограничивает разрешённые домены | 🟢 **Защищено** | Настроено в `main.py` |
| **Неверная валидация** | Pydantic модели с строгой типизацией | 🟢 **Защищено** | Отклоняет некорректные типы данных |
| **Race Conditions** | Unique Constraints в БД + транзакции | 🟢 **Защищено** | Гарантия уникальности пар валют |
| **Переполнение буфера** | Типизация Python + ограничения Pydantic | 🟢 **Защищено** | Строгие лимиты на длину строк |

## 3. Валидация входных данных

Все входящие данные проходят строгую проверку через Pydantic-схемы (`app/schemas.py`).

### Примеры правил валидации:

1.  **Код валюты:**
    *   Только 3 заглавные латинские буквы.
    *   Регулярное выражение: `^[A-Z]{3}$`.
    *   *Пример ошибки:* `us` (мало букв), `USD1` (цифра), `usd` (строчные).

2.  **Сумма конвертации:**
    *   Только положительные числа (`gt=0`).
    *   Тип `Decimal` (не Float) для избежания ошибок округления.

3.  **Курс обмена:**
    *   Положительное число.
    *   Точность до 6 знаков после запятой.

```python
# Пример из кода (app/schemas.py)
@field_validator('code')
@classmethod
def validate_code(cls, v: str) -> str:
    v = v.upper().strip()
    if not v.isalpha() or len(v) != 3:
        raise ValueError('Код валюты должен быть 3 заглавные буквы')
    return v
